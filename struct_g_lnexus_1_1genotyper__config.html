std::string GLnexus::genotyper_config::unrepr_symbolic_allele

Sometimes the input gVCF records contain specific allele calls that GLnexus is unable to render accurately in the output VCF file, e.g. rare alleles that are "pruned" during unification to ameliorate combinatorial explosion of the possible genotypes. By default, GLnexus emits no-calls (.) in such situations, which is the same thing that'd appear if the sample had no overlapping gVCF data at all. Sometimes it might be useful to distinguish these cases. If unrepr_symbolic_allele is set to a symbolic allele such as "?" or "NON_REF", then GLnexus will add this symbolic allele to the ALT of all output records, and calls of it indicate an unrepresentable allele call as described above. The symbolic allele must not contain angle brackets or whitespace.
